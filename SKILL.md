# OpenClaw 人设编辑器 🎭

一个通用工具，用于创建和优化 OpenClaw AI 助手的完整身份设定。

**核心特性：**
- 🎭 **真正的对话式交互** — 每次只问一个问题，等待用户回答
- 🧠 **智能更新机制** — 只修改身份相关字段，完全保留用户自定义内容
- 🔒 **权限请求** — 明确告知将要修改什么，等待你的确认
- 📝 **预览机制** — 显示所有将要进行的更改
- 🎨 **支持任意类型** — 动物、机器人、角色、概念...任何你能想到的

## 📋 对话流程

### 阶段一：准备阶段

**1. 权限请求**

当用户触发此技能时，首先请求权限：

> 🎭 我即将更新以下文件：
> - `~/.openclaw/workspace/IDENTITY.md`
> - `~/.openclaw/workspace/SOUL.md`
> - `~/.openclaw/workspace/USER.md`
> 
> 我会：
> - ✅ 更新：身份相关信息（名字、性格、世界观...）
> - ✅ 保留：你原有的其他所有内容
> - ❌ 不碰：与本次编辑无关的内容
> 
> **确认允许我修改这些文件吗？**
> （等待用户回复 "y" 或 "yes"）

> - E. 🎨 其他创意类型

**用户的回答示例：**
- "A"
- "动物类"
- "我想创建一个猫咪助手"

## 🔧 实现细节

### 对话模式的核心

**关键原则：**
- ✅ **每次只问一个问题**
- ✅ **等待用户回答后再问下一个**
- ✅ **不要假设或猜测用户的意图**
- ✅ **不自动填任何默认值**

### 数据收集方式

**使用临时变量存储：**
- 在对话过程中，收集到的数据存储在内存中
- 每个维度收集后，确认一次
- 最终一次性传递给脚本

**示例：**
```
# 收集基本信息
basic_info = {
    "name": "小煤球",
    "nickname": "煤球",
    ...
}

# 收集性格特质
personality = {
    "core_traits": "温和友善",
    ...
}

# 合并所有数据
persona_data = {**basic_info, **personality, ...}
```

## 🎯 关键要点

1. **不要批量收集信息** — 一次只问一件事
