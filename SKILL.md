# ⚡ 模式判断（第一步，必须先执行）

**AI 助手在执行任何操作前，必须先判断用户意图：**

## ✅ 快速优化模式（不询问权限）

**触发词：**
- "更X"（更可爱、更专业、更幽默、更活泼、更冷静...）
- "修改性格"、"调整说话"、"更新设定"、"优化设定"
- "改个名字"、"改昵称"、"换个名字"
- 任何包含"更新/修改/调整/优化/改" + 形容词/名词的表达

**流程：**
```
判断维度 → 读取当前值 → 给出 3-4 个建议选项 → 用户确认 → 更新文件
```

## 🆕 创建模式（需要权限）

**触发词：**
- "创建 AI 动手"、"定义新的人设"
- "创建角色"、"设计角色"
- "设计助手个性"、"设计角色背景"
- "开始创建"

**流程：**
```
请求权限 → 逐步收集 6 个维度 → 生成 800+ 字完整档案 → 用户确认 → 更新
```

**⚠️ 重要：如果用户说"更X"或"更新X"，必须走快速优化模式，不要询问权限！**

# OpenClaw 人设编辑器 🎭

创建和优化 OpenClaw AI 助手的完整身份设定。

---

## 📋 模式一：智能快速优化

### 核心原则
- **不询问权限**
- **主动判断用户意图对应的维度**
- **提供 3-4 个具体建议选项**
- **用户确认后直接更新**

### 维度判断映射

**性格相关：**
- "更可爱/活泼/有趣" → **核心特质**
- "更专业/可靠" → **核心特质**
- "更冷静/理性" → **核心特质**
- "更幽默/风趣" → **核心特质**
- "修改性格" → **核心特质**

**语言相关：**
- "调整说话/说话风格" → **表达方式**
- "调整语言风格" → **表达方式**
- "更简洁/更详细" → **表达方式**

**身份相关：**
- "改个名字/换个名字" → **名字**
- "改昵称" → **昵称**
- "修改世界观/背景故事" → **世界观**

### 示例流程

**用户说：** "更可爱一点"

**回应：**
> 🎭 我理解你想让助手更可爱一点。
> 
> **当前【核心特质】设定：**
> 温顺友善 — 平易近人、乐于助人
> 
> **建议修改为：**
> - A. 活泼可爱 — 有趣、精力充沛、惹人喜爱
> - B. 温顺可爱 — 安静、听话、惹人喜爱
> - C. 天真可爱 — 单纯、好奇、无忧无虑
> - D. 保持当前设定
> 
> **你觉得哪个更符合？**

## 📋 模式二：从零创建

### 流程

1. 请求权限
2. 6 个核心维度逐步收集（提供选项）
3. 生成 800+ 字完整角色档案
4. 用户确认完整档案
5. 模拟更新文件

### 6 个维度

1. **人设类型**（提供选项）
2. **基本信息**（名字/昵称自由输入，性别有选项）
3. **性格底色**（核心特质有选项）
4. **语言风格**（表达方式有选项）
5. **专属符号**（自由输入）
6. **世界观**（自由输入，多行）
